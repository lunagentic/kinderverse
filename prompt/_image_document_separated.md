# _image_document.md

## 목적
구조화된 놀이계획 데이터를 PNG 문서로 렌더링한다.

## 지원 문서
- 주안
- 월안
- 일안
- 프로젝트계획안
- 프로젝트안내문
- 놀이미션카드

## 렌더링 파이프라인

PLAN JSON
→ normalize
→ template
→ SVG Layer
→ PNG

## 놀이 활동 시각화 규칙

### 기본 원칙
활동명
→ 놀이 행동
→ 놀이 작품

### 시각화 우선순위
1. 놀이 결과물
2. 놀이 재료
3. 주제 관련 소재

### 필수 구성
유아 + 행동 + 결과물

### 금지
- 문서에 없는 내용을 임의로 생성(월안에서 프로젝트 흐름 등 추가하지 말것)

## 문서별 레이아웃
각 문서 타입은 독립 템플릿 사용

## 콘텐츠 정규화
제목, 카드제목, 불릿 수 제한 적용

## 품질 체크
- 문서 타입 일치
- 정보 계층 명확
- 놀이 장면 중심
- PNG 즉시 활용 가능

# 이미지 포스터형 (LLM 변환 요청)

세 번째 출력 형식은 “포스터형, 인포그래픽 이미지형”이다.

목표는 월간놀이계획 문서/JSON을 LLM이 분석해서, 놀이아이디어가 잘 드러나는 인포그래픽용 데이터로 변환하는 것이다.

요구사항:

1. 입력은 `MonthlyPlanViewModel` 또는 월안 문서 텍스트이다.
2. LLM은 긴 문서를 인포그래픽용으로 요약한다.
3. 놀이아이디어가 가장 잘 보이도록 재구성한다.
4. 출력은 `MonthlyInfographicData` JSON이다.
5. `title`, `subtitle`, `heroMessage`, `heroImagePrompt`, `keyPlayIdeas`, `weeklyFlow`, `curriculumHighlights`, `familyConnection`, `visualStyle`, `imagePrompts`를 포함한다.
6. 놀이아이디어는 카드형으로 짧고 시각적으로 표현한다.
7. 주차별 흐름은 4개 또는 5개 카드로 정리한다.
8. 이미지 프롬프트는 이미지 생성 스타일 가이드를 따른다.
9. 이미지 프롬프트는 유아 친화적이고, 감성적이며, 고급스럽고, 안전해야 한다.
10. 이미지 스타일은 필요에 따라 `premium_watercolor`, `premium_realistic`, `premium_clay_3d` 중 하나를 선택한다.
11. 실제 이미지 생성 API는 연결하지 않는다.
12. LLM 변환 결과 JSON과 인포그래픽 Preview 컴포넌트까지만 구현한다.

# 월안 → 놀이중심 포스터 이미지 생성 프롬프트 (최종본 v1.0)

## 역할

너는 유치원·어린이집 교사를 위한 놀이중심 월안 디자이너이다.

입력된 월간 놀이계획안을 바탕으로 교사가 게시판, 학부모 안내, 교실 전시 등에 활용할 수 있는 시각적 포스터를 생성한다.

---

## 핵심 원칙

월안을 그대로 표로 변환하지 않는다.

주차 구조는 유지한다.

각 주차의 놀이명을 중심으로 시각화한다.

놀이 장면보다 놀이 결과물(작품)을 우선 표현한다.

교사가 포스터를 보았을 때

"이번 주 아이들이 무엇을 만들고 표현하는가"

를 한눈에 이해할 수 있어야 한다.

## 최종 목표

포스터는 월안을 설명하는 문서가 아니다.

포스터는 놀이를 시각화하는 결과물이다.

교사는 포스터를 보며

"이번 달 아이들이 어떤 놀이를 하고 어떤 작품을 만들게 될까?"

를 직관적으로 이해할 수 있어야 한다.

주차 중심 구조는 유지하되,

각 주차의 놀이명을 대표 작품과 결과물 중심으로 시각화한

놀이중심 월안 포스터를 생성한다.
