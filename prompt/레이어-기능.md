# 레이어 기능 (Layer Decomposition & Editing)

생성된 활동지  PNG **한 장(flat image)** 을 분석해서, 텍스트와 이미지 요소를 각각
**편집 가능한 레이어**로 분리해 주는 기능입니다. 재생성이 아니라, 이미 만들어진 이미지
자체를 픽셀 분석으로 분해합니다.

---

## 1. 한눈에 보기

```
활동지 생성(PNG)  ──▶  [▣ 레이어] 클릭  ──▶  이미지 분석/분해  ──▶  레이어 편집기
   server.js              WorksheetModal       core/decompose.js      LayerEditor.jsx
   /api/worksheet                               /api/worksheet/decompose
```

- **입력**: 완성된 활동지 PNG 1장 (`outputs/worksheet_*.png`)
- **출력**: 배경 1 + 이미지 요소 N + 텍스트 M개로 분리된 레이어 문서(JSON) + 잘라낸 PNG들
- **편집**: 이동 / 크기조절 / 텍스트 수정 / 표시·숨김 / 순서변경 / 삭제

---

## 2. 채택한 방식 — A안 (경량 로컬 분석)

| 방식 | 내용 | 채택 |
|------|------|:----:|
| **A. 경량 로컬** | `sharp`(픽셀) + `tesseract.js`(OCR) + 연결요소 라벨링 | ✅ |
| B. SAM/클라우드 | Segment-Anything, LaMa inpainting, 외부 API | ✗ |

A안 선택 이유: 외부 의존성·비용 없이 로컬에서 즉시 동작(~2–3초), 충분한 분리 품질.

---

## 3. 분해 파이프라인 (`core/decompose.js`)

`decomposeWorksheet({ filename, baseUrl, thresholds })` 가 다음 순서로 처리합니다.

1. **이미지 로드** — `sharp` 로 RGBA raw 버퍼 확보 (`W, H, ch=4`)
2. **배경색 추정** — 테두리 픽셀 평균으로 배경색 산출 (`estimateBackground`)
3. **전경 마스크** — 각 픽셀의 배경색 거리(`colorDist`)가 임계값 초과면 전경
4. **텍스트 검출 (OCR)** — `tesseract.js` (`kor`,`eng`), `{ blocks: true }`
   - `blocks[].paragraphs[].lines[]` 순회
   - 신뢰도(`conf ≥ 60`) + 의미 필터(`meaningful`: 한글/영숫자 3자 이상) + IoU 중복 제거
   - 각 텍스트 → `{ text, x, y, w, h, fontSize, color }` 레이어로 변환, 텍스트 영역 마스킹
5. **이미지 요소 분리** — `objMask = 전경 AND NOT 텍스트` → `dilate`(팽창)로 인접 픽셀 묶음
6. **연결요소 라벨링** — BFS(`connectedComponents`)로 덩어리 분리, 최소면적 미만 제거 후 큰 순 정렬
7. **깨끗한 배경(clean plate)** — 모든 전경 픽셀을 배경색으로 메워 `background.png` 저장
8. **요소별 투명 컷아웃** — 각 덩어리를 투명 PNG(`obj{n}.png`)로 잘라 저장

### 임계값(thresholds) 기본값

```js
{ fg: 60, minAreaRatio: 0.0015, dilate: 3, ocrConf: 60 }
```

| 키 | 의미 | 조정 효과 |
|----|------|-----------|
| `fg` | 배경/전경 색거리 임계 | ↑ 전경 덜 잡힘 / ↓ 노이즈 증가 |
| `minAreaRatio` | 요소 최소 면적 비율 | ↑ 작은 조각 제거 |
| `dilate` | 마스크 팽창 반경 | ↑ 가까운 조각이 한 덩어리로 합쳐짐 |
| `ocrConf` | OCR 신뢰도 하한 | ↑ 오인식 텍스트 제거 |

### 반환 문서 구조

```jsonc
{
  "canvas":  { "w": 1024, "h": 1536, "background": "rgb(250,244,230)" },
  "source":  "worksheet_가을-낙엽-탐험_gpt_....png",
  "counts":  { "images": 38, "texts": 1 },
  "layers": [
    { "id": "bg", "kind": "background", "src": ".../background.png", "x":0, "y":0, "w":1024, "h":1536 },
    { "id": "o1", "kind": "image", "name": "요소 1", "src": ".../obj1.png", "x":.., "y":.., "w":.., "h":.. },
    // ... 이미지 요소들
    { "id": "t0", "kind": "text", "text": "낙엽을 분류해 보세요", "x":.., "y":.., "w":.., "h":.., "fontSize":.., "color":"rgb(..)" }
  ]
}
```

## 4. 레이어 편집기 (`board/src/components/LayerEditor.jsx`)

konva 등 외부 라이브러리 없이 **DOM 절대좌표 + 포인터 이벤트**로 구현했습니다.

- **표시 비율**: `scale = DISPLAY_W(440) / canvas.w` 로 실제 좌표↔화면 좌표 환산
- **이미지 레이어**: `.le-obj` 절대배치 div + `<img>`, 선택 시 우하단 `.le-handle`(비율 유지 크기조절)
- **텍스트 레이어**: `.le-text` div, **더블클릭 → `contentEditable` → blur 시 커밋**
- **드래그 이동**: `onPointerDown`→`startMove`, 배경 레이어는 이동 잠금
- **레이어 패널**: 위 레이어가 목록 상단(`reverse()`), 항목마다
  - 👁 표시/숨김 토글
  - 종류 태그 `T`(텍스트) / `I`(이미지) / `B`(배경)
  - ▲ ▼ 순서 변경(z-order swap), ✕ 삭제(배경 제외)

## 6. 사용 흐름 (UI)

1. 카드에서 **✦ 활동지 만들기** → 연령·주제·모델 선택 → **✦ 만들기**
2. 생성된 활동지 이미지 아래 **▣ 레이어** 클릭
3. 분해된 배경·이미지 요소·텍스트가 편집기에 표시됨
4. 요소를 드래그해 이동, 모서리 핸들로 크기조절, **텍스트는 더블클릭해 편집**
5. 패널에서 표시/숨김·순서변경·삭제

> ⚠️ `▣ 레이어` 는 **새로 생성한 직후**에만 동작합니다(`result.filename` 필요).

## 8. 알려진 한계 (A안)

- **스타일화된 제목**(예: "가을 낙엽 탐험")은 글자가 그림처럼 렌더링돼 OCR이 못 읽고
  **이미지 요소로 남습니다.** 일반 폰트 안내문은 텍스트로 정상 분리됩니다.
- 색이 비슷해 맞닿은 요소는 한 덩어리로 합쳐질 수 있음 → `dilate`/`fg` 조정 필요.
- OCR 오인식 가능(예: "낙엽을"→"777") → 편집기에서 더블클릭으로 직접 수정 가능.
